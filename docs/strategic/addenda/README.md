# Strategy addenda

**Purpose.** Time-stamped analyses that inform Atelier's strategic positioning without modifying the canonical `STRATEGY.md`. Each addendum is a snapshot of inputs (competitive landscape scans, talks, papers, market signals) at a moment in time; STRATEGY.md absorbs the durable conclusions as it evolves through normal PR cadence.

**When to write an addendum vs. update STRATEGY.md directly.**

- **Addendum:** time-bound analysis that may be revised as the landscape moves; competitive scans; talk reviews; one-time deep-dives. Preserves the snapshot for future reference.
- **STRATEGY.md edit:** durable strategic claims that the team commits to going forward; these survive the source addendum becoming stale.

The pattern matches `docs/architecture/audits/` (point-in-time milestone audits) and `docs/testing/*-benchmark-plan.md` (point-in-time research plans). Each is a snapshot with a date; the canonical doc remains the synthesized current state.

**Naming convention.** `YYYY-MM-DD-<topic-slug>.md`. Date is the analysis date (not the publication date of inputs). Topic slug is short and descriptive.

**Inputs to STRATEGY.md.** When an addendum's conclusions warrant updates to STRATEGY.md (e.g., competitive positioning needs sharpening, a known-staleness item is now resolved), file the STRATEGY.md edit as a follow-up PR that cites the addendum. The addendum stays as the historical record of the analysis that motivated the change.
